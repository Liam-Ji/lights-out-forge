# 黑灯工厂场景方案

## 主场景

**园区视频事件检索与报告生成**

一句话需求：

> 做一个视频事件检索看板，输入“昨晚园区是否有人员进入重点区域”，系统自动检索事件、展示截图，并生成事件摘要报告。

## 展示对象

面向领导、客户、技术决策者和行业专家。这个场景需要同时满足：

- 领导能快速看懂结果。
- 客户能联想到园区、安防、管理和合规场景。
- 技术决策者能看到 AI 编排业务能力的价值。
- 行业专家能理解它可以扩展到更复杂的指挥、巡检和安防系统。

## 演示闭环

需求输入 → AI 理解并拆解 → 生成视频事件检索原型 → 展示事件结果 → 生成事件摘要报告

这个闭环刻意保持轻量，不在第一版引入实时摄像头、实时视频流、地图、工单和多方通信。

## 匹配鸑鷟能力

| 展示环节 | 对应鸑鷟能力 | 价值 |
|---|---|---|
| 自然语言检索视频事件 | 视频语义搜索与问答 | 体现“一句话查视频事件” |
| 查看事件画面或片段 | 视频播放与采集 | 增强可信度和可视化效果 |
| 生成事件摘要报告 | 深度研究报告生成 / 文档生成 | 形成可交付业务结果 |
| 展厅大屏呈现 | 全屏大屏展示 | 适合领导和客户观看 |

## 选择理由

相比实时安防看板，这个场景更适合作为第一版：

- 不依赖真实摄像头。
- 不依赖实时视频流。
- 不依赖地图和工单系统。
- mock/脱敏样例数据容易准备。
- 能清楚体现鸑鷟已有视频和文档能力。
- 后续可以自然扩展为园区安防、巡检/巡防或应急指挥。

## 第一版边界

第一版只证明一件事：

> AI 能基于鸑鷟框架和鸑鷟能力，围绕一个明确业务需求生成可演示原型。

暂不展开：

- MCP 工具定义
- 接口字段
- mock 数据结构
- 页面详细布局
- 真实部署方案
- 应急预案细节

## 后续扩展方向

| 扩展方向 | 说明 |
|---|---|
| 园区安防 | 加入实时视频分析、告警、地图点位和处置流程 |
| 巡检/巡防 | 加入巡检点位、风险识别、处置记录和巡检报告 |
| 应急指挥 | 加入资源调度、视频会议、预案检索和处置纪要 |
