# 黑灯工厂后续实施路线

## 当前状态

第一版主场景已确定为“园区视频事件检索与报告生成”。当前还不进入技术实现细节。

## 后续路线

### 1. 场景确认

确认以下内容：

- 主场景名称是否固定为“园区视频事件检索与报告生成”。
- 一句话需求是否适合对外讲解。
- 是否统一使用“脱敏样例数据”作为展示数据口径。
- 标准演示时长是否控制在 3-5 分钟。

### 2. 演示叙事设计

把主场景整理成一条清晰讲解线：

需求输入 → AI 拆解 → 生成原型 → 展示视频事件 → 生成摘要报告

这一阶段只设计观众看到什么、讲解者怎么讲，不设计接口和代码。

### 3. 技术验证设计

场景和叙事确认后，再展开：

- 鸑鷟框架模板和组件规范。
- OpenHands 如何驱动原型生成。
- 是否需要 MCP。
- mock/脱敏样例数据如何准备。
- 页面布局和交互细节。

### 4. 原型实现

技术验证通过后，再进入实现：

- 基于鸑鷟框架搭建原型。
- 接入或模拟视频事件检索能力。
- 展示事件结果。
- 输出事件摘要报告。

### 5. 展厅打磨

最后再做展厅层面的可靠性和观感：

- 大屏展示布局。
- 预录视频兜底。
- 预烘焙版本兜底。
- 网络和模型异常预案。
- 讲解话术和演练。

## 不在当前阶段做的事

- 不定义 MCP 工具。
- 不定义接口字段。
- 不准备 JSON mock 数据。
- 不画详细页面。
- 不写代码。
- 不做部署方案。

## 下一步

只需要确认两件事：

1. 主场景是否最终定为“园区视频事件检索与报告生成”。
2. 对外讲解的一句话需求是否采用当前版本。
