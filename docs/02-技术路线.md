# 技术路线

## 选型对象

本次选型不是“做一个静态演示页”，而是选择一个能接收业务需求、驱动代码生成、产出可点击业务原型的系统方案。

## 推荐选型

**鸑鷟 AI + OpenClaw + Codex + 在线大模型 API + 脱敏样例数据**

用鸑鷟 AI 能力在目标底座上生成数据模型、页面、工作流和权限配置，优先验证“需求到业务原型”的主链路。

注意：鸑鷟 AI 是业务底座和应用生成能力，不等于完整工程操作工具。OpenClaw 负责 Agent 操作入口，Codex 负责本地可控搭建、脚本、插件开发、私有能力接入和调试。

其他路线不作为主线，但保留各自价值：

- OpenClaw + Codex：用于不限技术栈的自由原型生成，不受鸑鷟重型框架约束；OpenClaw 负责 Agent 操作入口，Codex 负责本地可控搭建与开发。
- Open Design：用于快速探索更好看的展厅视觉原型、dashboard 和演示稿。
- OpenHands：用于后续代码级改造、插件开发、私有能力接入和工程化验证。

## 选择矩阵

| 维度 | 鸑鷟 AI + OpenClaw + Codex | OpenClaw + Codex | Open Design | OpenHands |
|---|---|---|---|---|
| 核心价值 | 在目标底座上生成业务原型 | 不限技术栈，快速生成可运行原型 | 快速生成好看的视觉原型 | 展示代码级开发和插件改造 |
| 过程展示 | 中等，需要设计鸑鷟 AI + OpenClaw + Codex 的演示节奏 | 强，Agent 操作和本地生成过程清晰 | 强，生成过程直观 | 强，工程过程透明 |
| 结果沉淀 | 最强，结果直接在鸑鷟中 | 中等，结果可运行但后续需迁移到鸑鷟 | 弱，需要迁移 | 中等，沉淀在代码层而非业务配置层 |
| 私有能力承接 | 强，适合对齐能力清单 | 中等，可先模拟或封装，后续再接入鸑鷟 | 弱，只能模拟 | 强，适合插件和接口开发 |
| 工作量与难度 | ★★★★★ 最高 | ★★★ 中等 | ★★ 低 | ★★★★ 中高 |
| 主要风险 | 重型框架限制多，环境、权限、插件和私有能力约束会放大实施复杂度 | 技术栈自由但后续迁移成本不确定 | 容易变成一次性视觉 Demo | 容易变成技术演示，业务观众理解成本高 |
| 结论 | 主线方案 | 自由原型方案 | 视觉辅助/兜底 | 工程辅助 |

结论：如果目标是最终沉淀到鸑鷟，第一阶段仍以鸑鷟 AI + OpenClaw + Codex 为主线；如果目标是先快速证明“AI 能生成原型”，OpenClaw + Codex 是更轻的自由原型方案；Open Design 用来快速探索视觉效果和演示稿；OpenHands 用来补足后续代码、插件和私有能力接入。

## 不建议当前采用

| 方案 | 原因 |
|---|---|
| 自研 Agent 平台 | 开发量过大 |
| 直接做 MCP | 当前主链路还未验证 |
| 纯静态页面 | 不能证明“AI 生成原型”的系统能力 |
| Aider CLI | 开发效率可以，但展厅可视化弱 |
| Cline / VS Code 插件 | 可备选，但展示控制不如专门的原型生成界面 |

## 实施步骤

1. 固定一句话需求、目标观众、演示时长和脱敏样例数据口径。
2. 准备鸑鷟测试环境，确认 AI Agent 能操作应用。
3. 用 OpenClaw 作为 Agent 操作入口，连接鸑鷟环境。
4. 用 Codex 连接鸑鷟 CLI 和 Skills，跑通本地可控搭建流程。
5. 用 OpenClaw + Codex 跑一次不限技术栈的自由原型生成，作为轻量对照方案。
6. 基于能力清单整理第一版私有能力口径，只做“可展示、可模拟、可后续接入”的最小集合。
7. 用标准需求生成数据模型、页面和报告输出原型，验证浏览器预览和点击链路。
8. 用 Open Design 探索更好的视觉风格，必要时反向沉淀到鸑鷟页面设计。
9. 用 OpenHands 辅助插件开发、私有能力接入和工程化验证。
10. 录制成功路径并保留预烘焙版本，作为现场兜底。

## 成功标准

- AI 能基于一句需求在鸑鷟中生成或配置出目标原型。
- 原型能在浏览器中打开并点击。
- 样例数据和私有能力口径能与能力清单对齐。
- OpenClaw 能稳定作为 Agent 操作入口。
- Codex 能稳定完成本地搭建和修改任务。
- OpenClaw + Codex 自由原型方案能作为轻量对照，不影响主线演示稳定性。
- 生成过程能支撑 3-5 分钟讲解。
- 失败时可切到录屏或预烘焙版本。
- 后续能继续接入真实鸑鷟能力。

## 当前不做

- 不定义 MCP 工具。
- 不定义接口字段。
- 不接真实视频系统。
- 不做生产部署。

## 备注

鸑鷟基于 NocoBase。正文统一使用“鸑鷟”指代目标业务底座。
